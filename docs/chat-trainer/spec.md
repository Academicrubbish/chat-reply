# 聊天模拟器（Chat Simulator）Spec

## 背景

在社交聊天场景中，很多人不知道如何回复异性的消息，容易错过好感信号或回复不当。用户目前通过手动将聊天记录粘贴到 AI 工具中获取建议，操作繁琐且效率低。

本项目旨在开发一个**交互式聊天回复训练工具**，模拟真实聊天场景（微信风格），AI 实时分析对方信号并基于四大聊天策略法则生成多个回复选项，帮助用户在实战中练习和提升沟通技巧。

产品形态为本地 Web 应用，技术栈为 React 18 + TypeScript + Vite（前端）、Express（后端）、Claude API（AI 接口），纯 CSS 样式。

## 目标

构建一个左右分栏的本地 Web 应用：左侧为策略控制台（信号分析 + 策略建议 + 回复选项卡片），右侧为微信风格聊天模拟器。用户输入对方消息后，AI 实时分析信号并生成 1-5 条基于四大法则的回复选项，用户点选后自动上屏，形成闭环训练体验。

## 功能描述

### F1 - 女方消息输入
- 右侧面板底部提供输入框，placeholder 为"输入她刚发的消息..."
- 支持多行输入，最小高度 40px，最大高度 100px
- 发送方式：点击"发送"按钮
- 发送后消息立即以白色气泡（靠左）上屏到聊天区域
- 状态切换：WAITING_HER_MESSAGE → GENERATING_REPLIES

### F2 - AI 回复生成
- 后端接收完整对话历史，拼接系统提示词（四大法则），调用 Claude API
- 每次生成 1-5 条回复选项，每条标注策略名称和推荐理由
- 各条回复之间有明显的风格差异，至少包含一条安全型回复
- 回复自然、口语化，长度与对方发言匹配
- 生成期间显示"AI 分析中..."加载状态

### F3 - 信号分析展示
- 左侧面板顶部：情境状态栏，显示关系阶段标签（如"💕 暧昧期"）、场景标签（如"🌙 深夜聊天"）、信号标签（如"🟢 正面无冲突"）、推荐策略标签（如"✨ 魔趣法则"）、好感度进度条
- 左侧面板中部：Tab 切换区
  - [信号分析] Tab：AI 对对方消息的解读文本 + 情绪标签（正面/自嘲/延续话题等）+ 💡小贴士
  - [策略建议] Tab：推荐策略名称 + 核心思想 + 适用场景说明

### F4 - 回复选择
- 左侧面板下部：垂直排列的回复选项卡片列表
- 每张卡片包含：策略标签（如 [魔趣法则]）、回复文本（加粗）、推荐理由（灰色小字）
- 卡片策略标签颜色编码：魔趣法则=蓝色、平衡艺术=绿色、扩大冲突=粉色、安全回复=紫色
- Hover 交互：卡片上浮（translateY -2px）+ 阴影加深 + 边框变色
- 点击后：选中高亮（左侧蓝色边框 + 浅蓝背景），回复文本以绿色气泡上屏到右侧聊天区
- 状态切换：WAITING_SELECTION → WAITING_HER_MESSAGE

### F5 - 对话历史
- 右侧面板中部为消息流区域，垂直滚动，新消息在底部
- 对方消息：左侧靠齐，粉色渐变头像 + 白色气泡（圆角 12px，左上角直角）
- 我方消息：右侧靠齐，绿色渐变头像 + 绿色气泡（#95ec69，圆角 12px，右上角直角）
- 系统消息：居中灰色小字（如"—— 第3周 周三 ——"）
- 气泡最大宽度 70%
- 支持至少 50 条消息的连续对话

### F6 - 重置对话
- 右侧面板 Header 区域提供"重新开始"按钮
- 点击后清空所有聊天记录和左侧分析内容，回到初始状态

### UI 整体布局规格
- 整体背景：浅灰白 #f7f8fa
- 顶部标题栏：白色背景，左侧 Logo（渐变紫蓝圆角方块）+ 标题"聊天模拟器" + 副标题
- 左侧面板：40% 宽度（min-width: 400px, max-width: 520px），白色背景，右侧 1px #e8e8e8 边框
- 右侧面板：flex: 1（约 60%），灰色背景 #ededed
- 主按钮：深蓝 #3b5998，圆角 6px
- 字体：系统默认无衬线（-apple-system, PingFang SC, Microsoft YaHei）

## 边界条件

- **网络依赖**：需要网络连接调用 Claude API，离线状态下无法生成回复
- **API Key 安全**：ANTHROPIC_API_KEY 仅存在本地 .env 文件，不暴露到前端，后端作为代理转发请求
- **浏览器兼容**：仅支持 Chrome / Edge 最新版
- **对话长度限制**：支持至少 50 条消息的连续对话
- **响应速度**：AI 生成回复依赖 Claude API，目标 < 5 秒
- **输入校验**：空消息不可发送，发送按钮需在输入框有内容时可点击
- **API 失败处理**：Claude API 调用失败时，显示错误提示信息，用户可重试
- **并发限制**：同一时刻只允许一次 API 请求，生成中禁止重复发送
- **无持久化**：页面刷新后对话记录丢失（本地 Web 应用，无数据库）

## 验收标准

### VC1 - 消息输入与上屏
- 输入对方消息点击发送后，消息以白色气泡出现在聊天区左侧，输入框清空
- 空消息无法发送
- 多行文本正确显示

### VC2 - AI 回复生成
- 发送消息后，左侧面板显示"AI 分析中..."加载状态
- 加载结束后，左侧面板显示信号分析内容（情绪标签 + 解读文本 + 小贴士）
- 下方出现 1-5 张回复选项卡片
- 每张卡片有策略标签、回复文本、推荐理由三个要素
- 至少包含一张"安全回复"标签的卡片

### VC3 - 回复选择与上屏
- 点击某张回复卡片后，该回复以绿色气泡出现在聊天区右侧
- 左侧面板重置为"等待输入"状态
- 其他卡片取消选中态

### VC4 - 信号分析展示
- 情境状态栏正确显示：关系阶段标签 + 场景标签 + 信号标签 + 推荐策略标签
- 好感度进度条随 AI 分析结果更新
- Tab 切换在"信号分析"和"策略建议"之间正常工作

### VC5 - 对话历史
- 聊天区正确显示完整对话历史（对方白泡/我方绿泡/系统消息）
- 50 条消息内滚动流畅
- 新消息出现时自动滚动到底部

### VC6 - 重置对话
- 点击"重新开始"按钮后，聊天区和左侧面板全部清空
- 回到初始等待输入状态

### VC7 - 错误处理
- API Key 未配置时，显示明确错误提示
- 网络异常时，显示错误提示并提供重试
- 生成中重复点击发送，不触发重复请求
