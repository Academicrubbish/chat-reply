# V2 功能检查清单

> 启动方式：后端 `cd chat-reply-server && npm run dev`，前端 `cd chat-reply-trainer && npm run dev`
>
> 前置准备：确保 `.env` 配置了 `ZHIPU_API_KEY`，首次访问会进入初始化页

---

## Phase 1：安全加固

- [ ] **CORS 限制**：用外部工具（如 Postman）直接请求 `http://localhost:3001/api/targets`，不带 Origin 或带非白名单 Origin，应被拒绝
- [ ] **请求体限制**：发送超过 100KB 的 JSON body，应返回 413
- [ ] **限流**：1 分钟内快速发送 100+ 次请求，应返回 429
- [ ] **JSON 兜底**：观察 AI 生成时如果返回异常 JSON，前端应显示降级提示而非白屏
- [ ] **LLM 重试**：网络抖动时 AI 生成应自动重试，最终成功或显示兜底回复
- [ ] **ErrorBoundary**：正常使用时页面不受影响；故意制造渲染错误时显示错误提示 + 重试按钮

## Phase 2：登录认证

- [ ] **注册页**：点击登录页底部「注册」→ 进入注册页，输入用户名+密码注册，成功后跳转主页
- [ ] **注册校验**：空用户名、密码<6位、两次密码不一致、用户名已重复 → 均应提示错误
- [ ] **登录页**：输入正确凭据登录 → 跳转主页
- [ ] **登录校验**：错误密码 → 提示「用户名或密码错误」
- [ ] **Token 持久化**：登录后刷新页面 → 保持登录状态，不跳转登录页
- [ ] **Token 过期**：24h 后 Token 过期 → 自动跳转登录页（可用修改 JWT_SECRET 模拟）
- [ ] **未登录拦截**：直接访问 `/api/targets` 不带 Token → 返回 401

## Phase 3：体验优化

- [ ] **删除消息确认**：hover 消息 → 点删除图标 → 弹出 Popconfirm 气泡 → 取消不删/确认才删
- [ ] **删除对象确认**：下拉列表点删除 → 弹出 Modal「不可恢复」→ 取消不删/确认才删
- [ ] **引导页**：删除所有聊天对象 → 显示 3 步引导页 + 「开始使用」按钮 → 创建对象后引导页消失
- [ ] **性能**：50+ 条 AI 消息时操作流畅，无明显卡顿

## Phase 4：移动端 + 多模型

- [ ] **移动端 Tab**：Chrome DevTools 切到 iPhone SE（375px）→ 显示 [聊天] [AI辅助] Tab，可切换
- [ ] **移动端功能**：移动端下发送消息、AI 生成、选择回复均正常
- [ ] **模型选择器**：SessionBar 区域显示模型下拉（需 `.env` 配置至少一个模型 API Key）
- [ ] **模型切换**：切换模型后 AI 生成使用对应模型（检查后端日志确认 provider）
- [ ] **模型列表**：`GET /api/models` 仅返回已配置 API Key 的模型
