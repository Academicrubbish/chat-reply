# 聊天模拟器（Chat Simulator）需求文档

## 一、项目背景

在社交聊天场景中，很多人不知道如何回复异性的消息，容易错过好感信号或回复不当。用户目前通过手动将聊天记录粘贴到 AI 工具中获取建议，操作繁琐且效率低。

本项目旨在开发一个**交互式聊天回复训练工具**，模拟真实聊天场景，AI 实时分析对方信号并生成多个回复策略选项，帮助用户练习和提升沟通技巧。

---

## 二、产品定位

- **目标用户**：希望提升社交聊天能力的用户
- **核心价值**：将专业聊天策略（四大法则）融入实战练习，边聊边学
- **产品形态**：本地 Web 应用（React + Node.js）

## 五、界面设计 (UI/UX Design)

#### 5.1 整体布局架构 (Layout Structure)
采用**左右分栏布局**，适配桌面端浏览器，最大化利用屏幕空间。
*   **左侧面板 (策略与分析区):** 宽度占比约 40% (min-width: 400px)。负责展示背景信息、AI信号分析、以及回复策略选项。
*   **右侧面板 (聊天模拟区):** 宽度占比约 60% (flex: 1)。负责展示沉浸式聊天窗口，模拟真实社交软件（如微信）的视觉体验。
*   **背景色:** 整体背景采用浅灰白色 (`#f7f8fa`)，面板采用纯白 (`#ffffff`) 卡片式设计，带有轻微阴影。

#### 5.2 左侧面板：策略控制台 (Strategy Console)
参考截图中左侧的“故事/行程”面板，设计为信息聚合区。

*   **顶部：情境状态栏 (Context Bar)**
    *   **显示内容:** 当前关系阶段（如：暧昧期）、好感度进度条、当前时间/场景标签。
    *   **样式:** 类似截图顶部的 `Secret Romance • 第1周 周一`，使用胶囊标签 (Tag) 展示关键状态。
    *   **功能:** 让用户一眼看清当前的聊天背景。

*   **中部：信号分析区 (Signal Analysis)**
    *   **Tab切换:** 提供 `[信号分析]` `[策略建议]` 两个Tab（参考截图的 `故事 | 微博 | 豆瓣`）。
    *   **信号分析 Tab:**
        *   展示 AI 对对方上一句话的解读。
        *   **UI元素:** 使用彩色标签展示情绪（如 `🟢 正面` `🔴 冲突` ` 试探`）。
        *   **文本:** “对方在主动制造共同点，可以适当推进话题”（参考截图中的小提示）。

        *   展示当前的推荐策略名称（如 `推荐: 魔趣法则`）。

*   **下部：回复选项卡片 (Reply Options)**
    *   **布局:** 垂直列表排列，类似截图左侧的行程列表，但内容更丰富。
    *   **卡片内容:**
        *   **策略标签:** 左上角小标签（如 `[幽默]` `[真诚]`）。
        *   **回复文本:** 加粗显示，清晰易读。
        *   **推荐理由:** 灰色小字，解释为什么这么回（如“用幽默包装关心”）。
    *   **交互:** Hover 时卡片上浮并加深阴影；点击后直接发送并切换状态。

2.  **用户输入:** 用户在右侧底部输入框输入：“她：对啊，还是要起来当牛马”。
3.  **系统处理:**
    *   右侧聊天框立即上屏显示这条消息（白色气泡，靠左）。
    *   左侧面板刷新，显示“信号分析：抱怨工作，寻求安慰”。

5.  **反馈:**
    *   右侧聊天框显示“我”的回复（绿色气泡，靠右）。
    *   左侧面板重置，等待下一轮输入。

## 六、AI 回复生成规则

### 6.1 四大核心法则

系统内置四大聊天策略，AI 在生成回复时必须基于这些法则：

| 法则 | 核心思想 | 适用场景 |
|------|---------|---------|
| **扩大冲突** | 收到调侃后继续调侃，升级暧昧 | 对方发出好感/调侃信号时 |
| **魔趣法则** | 具有联系感的夸张行为，制造趣味 | 常规回应、需要制造趣味时 |
| **平衡艺术** | 一半赞美 + 一半抵消跪舔感 | 想表达兴趣但怕姿态太低时 |
| **释放性信息** | 恰到好处地表露对对方的兴趣 | 需要推进关系时 |

### 6.2 信号识别体系

AI 需要实时判断对方的态度信号：

| 信号类型 | 特征 | 应对策略 |
|---------|------|---------|
| 正面冲突 | 调侃、质疑、否定但语气轻松 | 扩大冲突 |
| 正面无冲突 | 主动分享、回复快、用疑问句延续 | 魔趣法则 |
| 模糊 | 回复不稳定、不置可否 | 安全回应 + 偶尔魔趣 |
| 负面 | 回复慢且简短、客气感谢 | 安全回应，保持距离 |

### 6.3 关系阶段判断

| 阶段 | 特征 | 适用策略 |
|------|------|---------|
| 初期接触 | 刚认识 | 魔趣法则（开场白） |
| 聊天升温 | 持续聊天有来有往 | 魔趣、平衡、适度释放 |
| 暧昧期 | 双方有调侃和暧昧信号 | 扩大冲突、释放性信息 |
| 约会/恋爱期 | 已在线下见面或确立关系 | 全部策略 |

### 6.4 回复生成要求

- 每次生成 1-5 条回复选项
- 每条回复标注使用的策略名称和推荐理由
- 各条回复之间有明显的风格差异
- 至少包含一条安全型回复（保底）
- 回复自然、口语化，像真人聊天
- 回复长度与对方发言匹配（对方短回复就不长篇大论）

## 七、数据结构

### 7.1 对话消息

```typescript
interface ChatMessage {
  role: 'her' | 'me';    // 消息发送者
  text: string;           // 消息内容
}
```

### 7.2 AI 返回结果

```typescript
interface GenerateRepliesResponse {
  analysis: {
    stage: string;        // 关系阶段
    signal: string;       // 对方信号
    strategy: string;     // 推荐策略
  };
  replies: Array<{
    id: number;           // 选项编号
    strategy: string;     // 策略名称
    reason: string;       // 推荐理由
    text: string;         // 回复文本
  }>;
  tip: string;           // 小提示
}
```

## 八、接口设计

### POST /api/generate-replies

**请求：**

```json
{
  "messages": [
    { "role": "her", "text": "对啊，还是要起来当牛马" },
    { "role": "me", "text": "销售果然是铁打的..." },
    { "role": "her", "text": "不会，这和天气没关系" }
  ]
}
```

**响应（成功）：**

```json
{
  "analysis": {
    "stage": "聊天升温",
    "signal": "正面无冲突",
    "strategy": "魔趣法则"
  },
  "replies": [
    {
      "id": 1,
      "strategy": "魔趣法则",
      "reason": "用幽默包装关心",
      "text": "害，我还以为你在雨里狂奔见客户呢，白替你担心了"
    },
    {
      "id": 2,
      "strategy": "平衡艺术",
      "reason": "关心+轻松转折",
      "text": "那心疼你算白心疼了，不过坐电脑前也累吧"
    }
  ],
  "tip": "对方在主动制造共同点，这是好感信号，可以适当推进话题"
}
```

**响应（失败）：**

```json
{
  "error": "ANTHROPIC_API_KEY 未配置"
}
```

## 十一、使用方式

1. 在 `.env` 文件中填入 `ANTHROPIC_API_KEY`
2. 运行 `npm run dev` 启动前后端
3. 浏览器打开 `http://localhost:5173`
4. 在输入框中输入女生的消息
5. 查看 AI 生成的回复选项，选择一条
6. 等待女生实际回复后，再次输入她的消息
7. 循环往复，练习沟通技巧
