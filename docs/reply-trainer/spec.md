# 聊天模拟器（Chat Simulator）一期 Spec

## 背景

在社交聊天场景中，很多人不知道如何回复异性的消息，容易错过好感信号或回复不当。用户目前通过手动将聊天记录粘贴到 AI 工具中获取建议，操作繁琐且效率低。

本项目旨在开发一个**交互式聊天回复训练工具**，模拟真实聊天场景（微信风格），支持多聊天对象管理，AI 实时分析对方信号并基于四大聊天策略法则（扩大冲突、魔趣法则、平衡艺术、释放性信息）生成多个回复选项。产品形态为本地运行的前后端分离 Web 应用，桌面端为主，数据通过 SQLite 持久化存储。

## 整体目标

构建一个左右分栏（左 70% AI Agent 控制台 + 右 30% 微信聊天模拟器）的本地 Web 应用。用户输入对方消息后，AI 基于人设信息、行为偏好和对话策略计划进行分析，生成 3-4 条风格各异的回复选项（含策略标签和推荐理由），用户点选或自定义回复后自动上屏，形成闭环训练体验。支持多聊天对象管理、AI 辅导窗口上下文隔离、对话策略动态规划、回复反馈机制等进阶功能。

## 功能描述

### F1 - 聊天对象管理

- 顶部栏右侧提供聊天对象选择器（下拉菜单），展示所有聊天对象列表（头像 + 名字 + 简介 + 消息数）
- 当前选中对象高亮，点击切换时右侧加载新对象聊天记录，左侧加载对应 AI 辅导窗口
- 下拉菜单底部提供「+ 新建聊天对象」入口，点击弹出创建 Modal
- 创建 Modal 表单字段：名字/称呼、认识场景、朋友圈人设/性格特点、兴趣爱好、近期聊天内容摘要
- 创建 Modal 包含 AI 行为偏好设置：语气偏好（aggressive 激进 / moderate 适中 / conservative 保守）、目标意图（practice 练习 / pursuing 追求 / friendship 友谊）、话题禁区（逗号分隔）
- 点击人设卡片「编辑人设」可重新打开 Modal 修改以上所有字段
- 下拉列表中每个对象提供删除入口，删除时清空该对象所有关联数据（聊天记录 + AI 窗口 + AI 消息）
- 无聊天对象时引导创建第一个

### F2 - 对话消息管理

- 右侧面板底部提供输入框，placeholder 为「输入她刚发的消息...」
- 支持多行输入，textarea 自适应高度（min 34px, max 80px）
- 发送方式：点击「发送」按钮
- 发送后消息以白色气泡（靠左）上屏到聊天区，来源标记为「手动输入」
- 每条消息记录来源：手动输入 / AI建议·策略名 / 自定义回复
- 右侧 Header 提供「重新开始」按钮，清空该对象的聊天记录和 AI 窗口
- 聊天记录持久化存储在 SQLite（chat_messages 表），按时间排序

### F3 - AI 辅导窗口管理

- 左侧面板人设卡片下方提供辅导窗口管理条，显示当前窗口编号（如 #3）和已用轮次
- 每个聊天对象可创建 N 个独立 AI 辅导窗口，每个窗口独立上下文
- 「窗口列表」按钮：下拉查看所有历史窗口，显示窗口编号和轮次数，点击切换
- 「+ 新窗口」按钮：创建全新 AI 对话上下文
  - 新窗口只携带人设信息 + 最近 10-15 条聊天消息（滑动窗口）
  - 不携带之前 AI 窗口的对话历史，防止上下文爆炸
  - 旧窗口保留可回看历史分析，但不再追加
- 上下文用量条：显示当前窗口估算 token 用量（百分比进度条）
- 上下文用量 >80% 时进度条变橙色，显示「建议新建窗口」提示文字

### F4 - AI 回复生成（核心接口）

- 触发方式：用户点击右侧 Header 的「AI 辅助」按钮
- 发送她消息后，右侧显示蓝色提示条「点击 AI 辅助获取回复建议」，带脉冲动画
- 后端处理流程：
  1. 收到请求，将 herMessage 存入 chat_messages（source: '手动输入'）
  2. 读取当前 session 的 ai_messages（构建多轮对话上下文）
  3. 读取该 target 的人设信息 + AI 行为偏好（语气/目标/禁区）
  4. 读取该 target 最近 10-15 条 chat_messages（滑动窗口）
  5. 读取当前 session 的 plan_goal + plan_next_step（对话策略计划）
  6. 估算当前上下文 token 数，更新 session.context_tokens
  7. 拼接系统提示词（四大法则 + 人设 + 行为偏好 + 对话计划 + 最近聊天 + 历史反馈偏好）
  8. 调用 GLM-5.1 API
  9. 解析返回的 JSON
  10. 将 AI 请求和响应存入 ai_messages
  11. session 的 round_count +1，更新 context_tokens
  12. 如 AI 返回了新的 plan_goal/plan_next_step，更新 session
  13. 返回分析结果 + 回复选项 + 对话计划 + token 用量
- 生成 3-4 条回复选项，风格各异，至少包含 1 条安全回复
- 回复自然口语化，长度与对方发言匹配
- 每条回复标注策略名和推荐理由
- 遵守用户设置的语气偏好和话题禁区
- 基于历史反馈偏好调整策略权重

### F5 - 回复选择与发送

- AI 生成完成后，弹出底部浮层弹窗（不占左侧面板空间）
- 弹窗内容：2x2 CSS Grid 四宫格回复卡片 + 自定义回复区
- 每张回复卡片：策略标签（颜色编码）+ 回复文本（加粗）+ 推荐理由 + 👍/👎反馈按钮
- 卡片策略标签颜色：魔趣法则=蓝色、平衡艺术=绿色、扩大冲突=粉色、安全回复=紫色
- 用户点击卡片选中高亮，再次点击确认选择 → 回复以绿泡上屏到右侧（来源标记：AI建议 · 策略名）→ 弹窗自动关闭
- 弹窗操作按钮：「🔄 重新生成」（全部换一批回复）、「🎯 指定策略」（下拉选择策略名重新生成）
- 自定义回复区：输入框 + 「发送自定义」按钮 → 自定义内容以绿泡上屏（来源标记：自定义回复）→ 弹窗自动关闭
- 弹窗右上角 ✕ 或点击遮罩层可关闭弹窗（暂不选择回复）
- 回复选择后，弹窗关闭，左侧全部空间还给信号分析

### F6 - 反馈机制

- 每张回复卡片提供 👍/👎 反馈按钮（不影响选择流程）
- 点击 👍/👎 后高亮对应按钮，记录反馈
- 后端存储反馈数据，后续 AI 生成时在 prompt 中加入历史偏好：「用户之前对平衡艺术策略反馈较好，对安全回复策略反馈较差」
- 反馈数据影响后续生成策略权重

### F7 - 信号分析展示

- 左侧面板显示 AI 分析结果：
  - **情境状态栏**：关系阶段标签（初期接触/聊天升温/暧昧期/约会恋爱期）、信号类型标签（正面冲突/正面无冲突/模糊/负面）、推荐策略标签、好感度进度条（0-100）
  - **对话策略计划卡片**：当前目标（如「升温关系 → 从暧昧期推进到约会」）+ 下一步建议（如「找机会发起线下邀约」），用户可手动编辑
  - **Tab 切换区**：
    - [信号分析] Tab：AI 对对方消息的解读文本 + 情绪标签（正面/自嘲/延续话题等）+ 💡小贴士
    - [策略建议] Tab：推荐策略名称 + 核心思想 + 适用场景说明
- AI 每次分析后更新：情境状态栏标签、好感度、对话策略计划

### F8 - AI 工作步骤进度

- AI 分析过程中，左侧面板显示步骤进度条（AgentSteps 组件）：
  - ✓ 分析对方消息（已完成）
  - ✓ 识别情绪信号（已完成）
  - ◐ 匹配策略知识...（进行中，脉冲动画）
  - ○ 生成回复选项（待执行）
- 步骤完成后实时更新状态（已完成/进行中/待执行）

### F9 - UI 整体布局

- 整体背景：浅灰白 #f7f8fa
- 顶部标题栏（高度 56px）：白色背景，Logo（渐变紫蓝圆角方块 + "AI"）+ 标题「聊天模拟器」+ 副标题 + 聊天对象选择器
- 左侧面板（70%，min-width 600px）：白色背景，包含人设卡片 → 辅导窗口管理条 → 情境状态栏 → 对话策略计划 → Tab（信号分析/策略建议）→ AI 工作步骤进度
- 右侧面板（30%，min-width 320px, max-width 420px）：灰色背景 #ededed，微信风格聊天模拟器
  - Header：头像 + 名字 + 在线状态 + 「AI 辅助」按钮 + 「重新开始」按钮
  - 消息流：垂直滚动，新消息在底部，自动滚动
  - 等待提示条：蓝色背景 + 脉冲动画 + 提示文字
  - 输入区：textarea + 发送按钮
- 回复选项弹窗：固定定位，从底部弹出，遮罩层背景模糊
- 新建/编辑聊天对象 Modal：居中弹窗，含完整表单
- 主色调：深蓝 #3b5998，字体：系统默认无衬线

### F10 - 数据持久化

- SQLite 数据库（better-sqlite3），4 张表：
  - `chat_targets`：聊天对象（id, name, meet_scene, persona, hobbies, recent_chats, tone_level, goal_intent, forbidden_topics, created_at）
  - `chat_messages`：聊天消息（id, target_id, role, text, source, strategy, session_id, created_at）
  - `ai_sessions`：AI 辅导窗口（id, target_id, title, is_active, round_count, context_tokens, plan_goal, plan_next_step, created_at）
  - `ai_messages`：AI 辅导消息（id, session_id, role, content, created_at）
- 数据关系：ChatTarget → ChatMessage[]（1 条连续聊天记录）+ AISession[]（N 个窗口）→ AIMessage[]（窗口内请求+响应）
- 页面刷新后数据不丢失

### F11 - 前后端通信

- 前端 Vite 开发服务器配置代理：`/api` → `http://localhost:3001`
- 后端端口 3001，前端端口 5173（Vite 默认）
- 后端 API 路由：
  - 聊天对象 CRUD：GET/POST `/api/targets`、GET/PUT/DELETE `/api/targets/:id`
  - 聊天消息：GET/POST/DELETE `/api/targets/:id/messages`
  - AI 辅导窗口：GET/POST `/api/targets/:id/sessions`、GET `/api/sessions/:sessionId/messages`
  - 核心生成：POST `/api/sessions/:sessionId/generate`
  - 回复操作：POST `/api/sessions/:sessionId/select-reply`、POST `/api/sessions/:sessionId/custom-reply`、POST `/api/sessions/:sessionId/regenerate`、POST `/api/sessions/:sessionId/feedback`
- GLM-5.1 通过 OpenAI SDK 兼容方式调用（baseURL: `https://open.bigmodel.cn/api/paas/v4/`）

## 边界条件

- **网络依赖**：需要网络连接调用 GLM-5.1 API，离线状态下无法生成回复（但本地数据可正常查看）
- **API Key 安全**：ZHIPU_API_KEY 仅存在后端 .env 文件，不暴露到前端，后端作为代理转发请求
- **浏览器兼容**：桌面端为主，Chrome / Edge 最新版，可做移动端适配
- **对话长度**：单次传给 AI 的滑动窗口为最近 10-15 条聊天消息，右侧聊天记录无上限
- **AI 上下文管理**：每个辅导窗口独立上下文，上下文用量 >80% 时建议新建窗口；新窗口只带人设 + 最近消息
- **响应速度**：AI 生成回复依赖 GLM-5.1 API，目标 < 5 秒
- **输入校验**：空消息不可发送
- **API 失败处理**：GLM-5.1 调用失败时显示错误提示，用户可重试
- **并发限制**：同一时刻只允许一次 AI 生成请求，生成中禁止重复触发
- **数据安全**：所有数据本地 SQLite 存储，无云同步
- **AI 行为约束**：遵守用户设置的语气偏好和话题禁区，基于历史反馈调整策略

## 验收标准

### VC1 - 聊天对象管理
- 可创建、编辑、删除聊天对象，表单包含所有字段（人设 + AI 行为偏好）
- 顶部选择器可切换对象，切换后左右面板数据正确刷新
- 无对象时显示创建引导

### VC2 - 对话消息输入与上屏
- 输入对方消息点击发送后，消息以白色气泡出现在聊天区左侧，来源标记「手动输入」
- 空消息无法发送
- 多行文本正确显示

### VC3 - AI 回复生成
- 点击「AI 辅助」后，左侧显示 AI 工作步骤进度
- 生成完成后弹出回复选项弹窗（底部浮层）
- 弹窗包含 3-4 张回复卡片，每张有策略标签 + 回复文本 + 推荐理由 + 👍/👎按钮
- 至少包含一张「安全回复」标签的卡片
- 左侧面板同步更新：情境状态栏标签、好感度进度条、对话策略计划、信号分析内容

### VC4 - 回复选择与上屏
- 点击回复卡片选中 → 再次点击确认 → 回复以绿泡上屏到右侧（来源标记「AI建议 · 策略名」）→ 弹窗自动关闭
- 自定义回复以绿泡上屏（来源标记「自定义回复」）→ 弹窗自动关闭
- 重新生成/指定策略可换一批回复
- 关闭弹窗（✕ 或遮罩）暂不选择回复

### VC5 - AI 辅导窗口管理
- 可创建新窗口，新窗口上下文重置（只带人设 + 最近消息）
- 可切换回看历史窗口
- 上下文用量进度条正确显示，>80% 变橙色并显示「建议新建窗口」

### VC6 - 信号分析展示
- 情境状态栏正确显示：关系阶段 + 信号类型 + 推荐策略标签 + 好感度进度条
- Tab 切换在「信号分析」和「策略建议」之间正常工作
- 对话策略计划卡片显示当前目标 + 下一步建议，可手动编辑
- AI 每次分析后自动更新以上内容

### VC7 - 反馈机制
- 点击 👍/👎 按钮可记录反馈，按钮高亮
- 反馈数据持久化，后续 AI 生成考虑历史偏好

### VC8 - 数据持久化
- 页面刷新后聊天记录、人设信息、AI 窗口数据不丢失
- 切换聊天对象后数据正确恢复

### VC9 - 错误处理
- API Key 未配置时显示明确错误提示
- 网络异常时显示错误提示并可重试
- 生成中重复点击「AI 辅助」不触发重复请求
