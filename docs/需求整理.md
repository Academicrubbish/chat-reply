# 聊天回复训练器 — 一期需求（实现版）

> H5 Web 应用，桌面端为主，可做移动端适配。产品形态为本地运行的前后端分离应用。

---

## 一、项目架构

```
reply/
├── chat-reply-trainer/        # 前端（React 18 + TypeScript + Vite）
├── chat-reply-server/         # 后端（Express + SQLite + OpenAI SDK）
└── docs/                      # 文档
```

前后端分离，独立目录同级放置。前端 Vite 开发时代理 API 到后端。

**技术栈：**

| 层 | 技术 |
|----|------|
| 前端框架 | React 18 + TypeScript |
| 构建 | Vite |
| 后端 | Express |
| 数据库 | SQLite（better-sqlite3） |
| AI | 智谱 GLM-5.1（OpenAI SDK 兼容） |
| 样式 | 纯 CSS |

## 二、数据模型

### 2.1 聊天对象（ChatTarget）

```typescript
interface ChatTarget {
  id: string;            // UUID
  name: string;          // 名字/称呼
  meetScene: string;     // 认识场景
  persona: string;       // 朋友圈人设/性格特点
  hobbies: string;       // 兴趣爱好
  recentChats: string;   // 近期聊天内容摘要
  // --- AI 行为偏好（Harness: Permission） ---
  toneLevel: 'aggressive' | 'moderate' | 'conservative';  // 语气偏好
  goalIntent: 'practice' | 'pursuing' | 'friendship';     // 目标意图
  forbiddenTopics: string;  // 话题禁区（逗号分隔）
  createdAt: number;
}
```

SQLite 表：
```sql
CREATE TABLE chat_targets (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  meet_scene TEXT DEFAULT '',
  persona TEXT DEFAULT '',
  hobbies TEXT DEFAULT '',
  recent_chats TEXT DEFAULT '',
  tone_level TEXT DEFAULT 'moderate',
  goal_intent TEXT DEFAULT 'pursuing',
  forbidden_topics TEXT DEFAULT '',
  created_at INTEGER NOT NULL
);
```

### 2.2 聊天消息（ChatMessage）

每个聊天对象 1 条连续时间线，存储在右侧聊天窗口。

```typescript
interface ChatMessage {
  id: string;
  targetId: string;          // 绑定聊天对象
  role: 'her' | 'me';
  text: string;
  source: '手动输入' | 'AI建议' | '自定义回复';
  strategy?: string;         // AI建议时标记策略名
  sessionId?: string;        // 来自哪个AI辅导窗口
  createdAt: number;
}
```

SQLite 表：
```sql
CREATE TABLE chat_messages (
  id TEXT PRIMARY KEY,
  target_id TEXT NOT NULL,
  role TEXT NOT NULL,        -- 'her' | 'me'
  text TEXT NOT NULL,
  source TEXT NOT NULL,      -- '手动输入' | 'AI建议' | '自定义回复'
  strategy TEXT DEFAULT NULL,
  session_id TEXT DEFAULT NULL,
  created_at INTEGER NOT NULL,
  FOREIGN KEY (target_id) REFERENCES chat_targets(id)
);
CREATE INDEX idx_messages_target ON chat_messages(target_id, created_at);
```

### 2.3 AI 辅导窗口（AISession）

每个聊天对象可创建 N 个窗口，每个窗口独立上下文。

```typescript
interface AISession {
  id: string;
  targetId: string;
  title: string;             // 如 "#1"、"#2"
  isActive: boolean;
  roundCount: number;        // 已用轮次
  contextTokens: number;     // 当前上下文估算 token 数
  // --- 对话策略计划（Harness: TodoWrite） ---
  planGoal: string;          // 当前目标，如 "升温关系 → 从暧昧期推进到约会"
  planNextStep: string;      // 下一步建议，如 "找机会发起线下邀约"
  createdAt: number;
}
```

SQLite 表：
```sql
CREATE TABLE ai_sessions (
  id TEXT PRIMARY KEY,
  target_id TEXT NOT NULL,
  title TEXT NOT NULL,
  is_active INTEGER DEFAULT 1,
  round_count INTEGER DEFAULT 0,
  context_tokens INTEGER DEFAULT 0,
  plan_goal TEXT DEFAULT '',
  plan_next_step TEXT DEFAULT '',
  created_at INTEGER NOT NULL,
  FOREIGN KEY (target_id) REFERENCES chat_targets(id)
);
```

### 2.4 AI 辅导消息（AIMessage）

每个窗口内的请求+响应记录。

```typescript
interface AIMessage {
  id: string;
  sessionId: string;
  role: 'user' | 'assistant';
  content: string;           // JSON 字符串，包含分析/回复/请求等
  createdAt: number;
}
```

SQLite 表：
```sql
CREATE TABLE ai_messages (
  id TEXT PRIMARY KEY,
  session_id TEXT NOT NULL,
  role TEXT NOT NULL,        -- 'user' | 'assistant'
  content TEXT NOT NULL,     -- JSON
  created_at INTEGER NOT NULL,
  FOREIGN KEY (session_id) REFERENCES ai_sessions(id)
);
CREATE INDEX idx_ai_messages_session ON ai_messages(session_id, created_at);
```

### 2.5 数据关系

```
ChatTarget (聊天对象)
  ├── ChatMessage[]    1条连续聊天记录（右侧微信聊天）
  └── AISession[]      N个AI辅导窗口（左侧AI分析）
        └── AIMessage[]  窗口内的请求+响应
```

### 3.4 核心接口：POST /api/sessions/:sessionId/generate

**请求：**
```json
{
  "herMessage": "对啊，还是要起来当牛马"
}
```

**后端处理流程：**
1. 收到请求，将 herMessage 存入 chat_messages（source: '手动输入'）
2. 读取当前 session 的 ai_messages（构建多轮对话上下文）
3. 读取该 target 的人设信息 + AI行为偏好（语气/目标/禁区）
4. 读取该 target 最近 10-15 条 chat_messages（滑动窗口）
5. 读取当前 session 的 plan_goal + plan_next_step（对话策略计划）
6. 估算当前上下文 token 数，更新 session.context_tokens
7. 拼接系统提示词（四大法则 + 人设 + 行为偏好 + 对话计划 + 最近聊天）
8. 调用 GLM-5.1 API
9. 解析返回的 JSON
10. 将 AI 请求和响应存入 ai_messages
11. session 的 round_count +1，更新 context_tokens
12. 如 AI 返回了新的 plan_goal/plan_next_step，更新 session
13. 返回分析结果 + 回复选项 + 对话计划 + token 用量

**响应：**

### 3.5 用户选择/自定义回复

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/sessions/:sessionId/select-reply` | 用户选择AI回复 |
| POST | `/api/sessions/:sessionId/custom-reply` | 用户自定义回复 |
| POST | `/api/sessions/:sessionId/regenerate` | 重新生成（可选指定策略） |
| POST | `/api/sessions/:sessionId/feedback` | 对AI建议的反馈（👍/👎） |

**select-reply 请求：**
```json
{ "replyId": 1 }
```
后端：找到对应 reply，存入 chat_messages（role: 'me', source: 'AI建议', strategy: '魔趣法则'）

**custom-reply 请求：**
```json
{ "text": "我自己的回复内容" }
```
后端：存入 chat_messages（role: 'me', source: '自定义回复'）

**regenerate 请求（Harness: Error Recovery）：**
```json
{ "preferredStrategy": "扩大冲突" }
```
`preferredStrategy` 可选，不传则 AI 自由选择策略组合。后端在 ai_messages 末尾追加一条 user 消息："用户不满意上次的结果，请重新生成回复。用户偏好策略：扩大冲突"，然后重新调用 GLM-5.1。

**feedback 请求（Harness: 反馈回路）：**
```json
{ "replyId": 2, "rating": "thumbs_up" }
```
后端记录反馈（存入 ai_messages 或独立表），后续生成时在 prompt 中加入历史偏好："用户之前对平衡艺术策略的反馈较好，对安全回复策略反馈较差"。

## 五、交互流程

### 5.1 用户操作主流程

```
1. 用户打开应用
   → 如果没有聊天对象，引导创建第一个
   → 如果有，默认选中最近使用的对象

2. 选择/创建聊天对象后
   → 右侧加载该对象的聊天记录
   → 左侧加载/创建当前AI辅导窗口

3. 用户在右侧输入框输入她发的消息 → 点击「发送」
   → 消息以白泡出现在右侧聊天区（来源：手动输入）
   → 显示蓝色提示条「点击 AI 辅助获取回复建议」

4. 用户点击右侧「AI 辅助」按钮
   → 左侧展示 AI 工作步骤进度条（Harness: Agent Loop 透明）
     ┌─────────────────────────────────────┐
     │ ✓ 分析对方消息                       │
     │ ✓ 识别情绪信号                       │
     │ ◐ 匹配策略知识...                    │
     │ ○ 生成回复选项                       │
     └─────────────────────────────────────┘
   → 后端：保存消息 + 调用 GLM-5.1 API + 存AI记录 + 更新计划
   → 左侧更新：对话计划 + 信号分析 + 上下文用量
   → 弹出回复选项浮层（底部弹窗，不占左侧面板空间）

5. 用户选择回复（在弹出浮层中操作）
   A) 点击四宫格中某张卡片
      → 可对卡片点 👍/👎 反馈（反馈被记录，影响后续生成质量）
      → 再次点击确认选择 → 回复以绿泡上屏到右侧（来源：AI建议 · 策略名）
      → 弹窗自动关闭，左侧全部空间还给信号分析
   B) 点击「🔄 重新生成」
      → 全部换一批回复（Harness: Error Recovery）
   C) 点击「🎯 指定策略」→ 下拉选择策略名 → 按指定策略重新生成
   D) 在自定义回复区输入并发送
      → 自定义内容以绿泡上屏到右侧（来源：自定义回复）
      → 弹窗自动关闭
   E) 点击弹窗右上角 ✕ 或点击遮罩层
      → 关闭弹窗，暂不选择回复

6. 回到步骤 3，循环往复
```

### 5.2 辅助操作

- **切换聊天对象**：顶部栏下拉选择 → 右侧加载新对象聊天记录 → 左侧加载对应AI窗口
- **新建AI辅导窗口**：左侧点击「+新窗口」→ 全新上下文（只带人设+最近消息） → 旧窗口保留可回看
- **编辑人设+AI偏好**：点击「编辑人设」→ 弹出 Modal 修改人设 + 语气偏好/目标意图/话题禁区
- **对话计划**：AI 每次分析后更新「当前目标」和「下一步建议」，用户可手动编辑计划
- **上下文预警**：上下文用量 >80% 时，辅导窗口管理条变橙色并显示「建议新建窗口」提示
- **重新开始**：清空当前对象的聊天记录和AI窗口
- **删除对象**：在下拉列表中提供删除入口

## 六、AI 提示词设计

### 6.1 系统提示词结构

```
[角色定义]
你是一个社交聊天辅导AI，帮助用户分析对方消息并生成回复建议。

[四大法则]
1. 扩大冲突：收到调侃后继续调侃，升级暧昧
2. 魔趣法则：具有联系感的夸张行为，制造趣味
3. 平衡艺术：一半赞美 + 一半抵消跪舔感
4. 释放性信息：恰到好处地表露对对方的兴趣

[信号识别]
- 正面冲突：调侃、质疑 → 扩大冲突
- 正面无冲突：主动分享、疑问句延续 → 魔趣法则
- 模糊：不稳定、不置可否 → 安全回应 + 偶尔魔趣
- 负面：回复慢且简短 → 安全回应，保持距离

[关系阶段]
- 初期接触 / 聊天升温 / 暧昧期 / 约会恋爱期

[对方人设]
名字：{name}
认识场景：{meetScene}
性格/人设：{persona}
兴趣爱好：{hobbies}
近期聊天摘要：{recentChats}

[AI 行为偏好]
语气偏好：{toneLevel}  （aggressive激进 / moderate适中 / conservative保守）
目标意图：{goalIntent}  （practice只是练习 / pursuing正在追 / friendship维持友谊）
话题禁区：{forbiddenTopics}  （不要涉及这些话题）

[当前对话策略计划]
当前目标：{planGoal}
```

## 十一、GLM-5.1 接入方式

智谱 GLM-5.1 完全兼容 OpenAI API 格式，后端使用 `openai` npm 包即可调用。

**安装依赖：**
```bash
npm install openai
```

**后端初始化示例（chat-reply-server/src/llm.ts）：**
```typescript
import OpenAI from 'openai';

const client = new OpenAI({
  apiKey: process.env.ZHIPU_API_KEY,
  baseURL: process.env.ZHIPU_BASE_URL || 'https://open.bigmodel.cn/api/paas/v4/',
});

const MODEL = process.env.ZHIPU_MODEL || 'glm-5.1';

export async function chatCompletion(messages: Array<{role: string; content: string}>) {
  const response = await client.chat.completions.create({
    model: MODEL,
    messages,
    temperature: 0.8,
    max_tokens: 2048,
  });
  return response.choices[0].message.content;
}
```

**调用示例（在 generate 接口中）：**
```typescript
// 构建消息
const systemPrompt = buildSystemPrompt(target, recentMessages);
const conversationMessages = loadSessionMessages(sessionId);

const messages = [
  { role: 'system', content: systemPrompt },
  ...conversationMessages,
  { role: 'user', content: `对方的最新消息是：${herMessage}` }
];

// 调用 GLM-5.1
const raw = await chatCompletion(messages);

// 解析 JSON
const result = JSON.parse(raw);
return result; // { analysis, replies }
```

**关键信息：**
- API 端点：`https://open.bigmodel.cn/api/paas/v4/chat/completions`
- 鉴权方式：`Authorization: Bearer YOUR_API_KEY`
- 模型名：`glm-5.1`
- 完全兼容 OpenAI SDK，只需改 `baseURL` 和 `apiKey`
- 官方文档：https://docs.bigmodel.cn/cn/guide/develop/openai/introduction
