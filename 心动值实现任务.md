# 心动值（Attraction Score）实现任务

## 背景

这是一个聊天回复助手应用（chat-reply），帮助用户模拟和优化与心仪对象的微信聊天。应用有两部分：
- `chat-reply-server/` — Express + SQLite 后端
- `chat-reply-trainer/` — React + Vite + Ant Design 前端

现在要实现**心动值**功能：对标 galgame 好感度系统，给用户一个直观的量化指标（0-100 分），告诉用户"对方对你有多心动"。诊断方案是定性分析（态度、情绪、阶段），心动值是定量分数。

### 关键设计决策

| 环节 | 更新心动值 | 理由 |
|---|---|---|
| 诊断方案（首次诊断） | ✅ 生成初始分数 + 依据 | 基于聊天记录综合分析 |
| AI 辅助—完整模式 | ✅ 每次更新分数 + 变化原因 | 分析最新消息的对方态度 |
| AI 辅助—快速模式 | ❌ 不更新 | 快速模式只生成回复文案 |
| 军师分析 | ❌ 不更新 | 独立分析工具 |
| 复盘 | ❌ 不更新 | 用户表现反思 |

心动值存储在 `chat_targets.attraction_score`（INTEGER），与诊断数据（`chat_diagnoses` 表）分开。

### 任务 6：创建 AttractionBar 组件

**新建文件**：`chat-reply-trainer/src/components/AttractionBar.tsx`

这是一个独立的心动值展示条，放在 Toolbar（session-bar）下方。组件规格：

```tsx
interface AttractionBarProps {
  attraction: { score: number; reason: string } | null;
  isMobile?: boolean;
}
```

- 当 `attraction === null` 时返回 `null`（不渲染）
- 左侧 4px 彩色竖条 + 心形图标 + 分数数字 + 原因说明文字
- 颜色：score < 30 → 灰色(#999)，30-59 → 橙色(#fa8c16)，60-79 → 粉色(#eb2f96)，≥80 → 红色(#f5222d)
- 背景色用对应颜色的 10% 透明度版本
- CSS transition 让颜色和文字平滑变化
- 桌面端一行显示，移动端可换行

**代码模板**：
```tsx
import React from 'react';
import { HeartFilled } from '@ant-design/icons';

interface AttractionBarProps {
  attraction: { score: number; reason: string } | null;
  isMobile?: boolean;
}

const getColor = (score: number) =>
  score >= 80 ? '#f5222d' : score >= 60 ? '#eb2f96' : score >= 30 ? '#fa8c16' : '#999';

const AttractionBar: React.FC<AttractionBarProps> = ({ attraction, isMobile = false }) => {
  if (!attraction) return null;
  const color = getColor(attraction.score);
  return (
    <div style={{
      display: 'flex', alignItems: isMobile ? 'flex-start' : 'center', gap: 8,
      padding: isMobile ? '6px 12px' : '8px 20px',
      borderLeft: `4px solid ${color}`,
      background: `${color}10`,
      borderBottom: '1px solid #ebeef5',
      flexShrink: 0,
      transition: 'all 0.5s ease',
      flexDirection: isMobile ? 'column' : 'row',
    }}>
      <div style={{ display: 'flex', alignItems: 'center', gap: 6, flexShrink: 0 }}>
        <HeartFilled style={{ color, fontSize: 16 }} />
        <span style={{ fontSize: 20, fontWeight: 700, color, lineHeight: 1 }}>
          {attraction.score}
        </span>
      </div>
      <span style={{
        fontSize: 12, color: '#666',
        flex: 1, minWidth: 0,
      }}>
        {attraction.reason}
      </span>
    </div>
  );
};

export default AttractionBar;
```

## 注意事项

1. **不要修改已存在的正确代码**：`runDiagnosisInternal`、诊断端点、DB migration、前端 types（除 TargetDiagnosis）、sseHandlers.ts 的处理逻辑都已是正确的
2. **快速模式（quick mode）不产生 attraction**：不要试图在 quick mode 中加回来
3. **全量模式 prompt 的 JSON 输出格式要精确**：LLM 会严格按照你给的 schema 输出，如果 schema 不对就提取不到
4. **`parsed.attraction` 是顶层字段**：新 prompt 的 attraction 在顶层，不是嵌套在 `analysis` 里
5. **心动值绑定在 target 上**：与 session 无关，切换窗口不影响
6. **如果 LLM 不返回 attraction 字段**：`attractionResult` 为 null，前端不显示，属于正常降级
