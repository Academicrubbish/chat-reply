# 聊天记录测试集

用于测试 chat-reply 系统 AI 回复质量和生成耗时的 8 个场景化测试数据。

## 快速开始

```bash
# 1. 启动后端服务器
cd chat-reply-server && npm run dev

# 2. 另一个终端，导入测试数据
node test-data/seed.js

# 3. 启动前端，在浏览器中测试
cd chat-reply-trainer && npm run dev
```

## 测试方法

### Full 模式（分析 + 回复生成）
1. 选择聊天对象 → 在右侧输入对方的最后一条消息
2. 点击 **AI辅助** 按钮
3. 观察分析结果（阶段、信号、策略）和回复建议
4. 检查是否引用了正确的知识单元 ID

### Quick 模式（快速回复）
1. 在 ChatHeader 切换到 **快速** 模式
2. 发送最后一条消息，点击 AI 辅助
3. 对比生成速度和回复质量

### Advisor 模式（军师分析）
1. 点击工具栏 **军师** 按钮
2. 检查诊断结果中的态度级别、语言模式、升级判断
3. 验证 warnings 是否正确触发

### Review 模式（复盘总结）
1. 先用 full 模式完成几轮对话并选择回复
2. 点击工具栏 **复盘** 按钮
3. 检查五维评分雷达图、亮点/踩坑分析、警告级别

## 性能基准

建议记录以下指标：

| 指标 | 说明 | 预期范围 |
|------|------|---------|
| TTFB | 首字节到达时间 | < 2s |
| 总生成时间 | 从点击到全部完成 | full: < 15s, quick: < 8s |
| 分析质量 | 信号识别准确率 | > 80% |
| 知识引用 | 正确引用知识单元 ID | 覆盖 expected.knowledgeIds |

## 文件格式

```json
{
  "name": "场景名称",
  "description": "场景描述",
  "target": { "name", "meet_scene", "persona", "hobbies", "tone_level", "goal_intent", "forbidden_topics" },
  "messages": [{ "role": "her|me|scene", "text": "...", "source": "手动输入|自定义回复|场景补充" }],
  "expected": { "stage": "...", "signal": "...", "knowledgeIds": [...], "tips": "..." }
}
```
